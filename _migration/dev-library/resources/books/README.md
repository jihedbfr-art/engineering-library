# Books

Not filled in yet — I keep starting this list and then just... not finishing it, because "the ones worth your time" is a much higher bar to write honestly than "here are some famous titles."

For now, three I'd actually defend without hesitation:
- **Designing Data-Intensive Applications** (Kleppmann) — if you only ever read one systems book, this is it.
- **The Pragmatic Programmer** — ages better than it has any right to.
- **Clean Architecture** (Uncle Bob) — take the dogma with a grain of salt, keep the underlying ideas about dependency direction.
