# Pourquoi pas de framework ?

Ce mini-projet évite volontairement Spring Boot pour rester **léger** et sans dépendance réseau.

## Les avantages

- Démarrage instantané
- Pas de `pom.xml` à résoudre
- Code entièrement lisible en une poignée de fichiers

## Les limites

- Pas de gestion avancée de routes
- Le rendu Markdown est un sous-ensemble volontairement simple

C'est un compromis assumé pour un budget de développement très court.
