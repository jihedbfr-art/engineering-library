# engineering-playbooks

Playbook = **procédure**. Le dossier qu'on ouvre à 2h du matin, réveillé, sans réfléchir : on suit les
étapes dans l'ordre. Rare et précieux.

## Structure d'un playbook
Quand le déclencher · Pré-requis · Étapes numérotées · Vérification · Communication · Post-mortem. Voir [`_TEMPLATE.md`](_TEMPLATE.md).

## Backlog
Incident production · Déploiement · Rollback · Hotfix · Migration · Panne base de données ·
Panne Kafka · Redis down · Incident de sécurité · Récupération de données.
Ajouter via la skill **ajouter-entree-savoir**.
