# code-review-guide

Le repo que tout Tech Lead devrait avoir. Comment reviewer : quoi regarder, dans quel ordre, et les
smells/anti-patterns à traquer. Un fichier par axe.

## Structure d'une entrée
Quoi chercher · Pourquoi ça compte · Exemple à rejeter · Exemple accepté. Voir [`_TEMPLATE.md`](_TEMPLATE.md).

## Axes
Comment reviewer (méthode) · Code smells · Anti-patterns · Performance (N+1, allocations) · Tests ·
Sécurité (injection, secrets, authz). Ajouter via la skill **ajouter-entree-savoir**.
